## Binary Tree

Un binary tree es una estructura de datos Tree que tiene que seguir unas ciertas reglas:

- Un nodo debe tener como maximo 2 nodos hijos
- Un hijo no puede tener mas de un padre

### Cuando Usarlo?

El binary tree como vemos, cuando es un perfect binary tree es la estructura de datos mas rapida en todas las operaciones. Tambien podemos ver que es ordenado. Por lo tanto es facil de imaginar donde está cada nodo y como esta montada la estructura. Esta tiene un tamaño flexible.

En general es bueno utilizarlo cuando tenemos datos que encajan en una Binary Tree balanceado o perfecto. (por ejemplo en una lista al azar o en algo relativo)

### Cuando no usarlo

Una de las desventajas es que ninguna operacion tiene O(1) de complejidad por lo que si solo vamos a utilizar una lista para añadir un elemento o eliminar un ultimo o primer elemento es mejor utilizar otra estructura de datos como linked list o Arrays

## AVL Trees + Red Black Trees

estos son muy similares y muy utilices para balancear un Binary Tree  ya que estos lo que hacen es cuando se añade o se elimina un elemento, cambiara el orden de los nodos para balancearlos. 

### AVL Tree

El árbol AVL toma su nombre de las iniciales de los apellidos de sus inventores, 

<a href="https://es.wikipedia.org/wiki/Georgii_Adelson-Velskii" class="link  white hover-white-60" target="_blank">Georgii Adelson-Velskii</a> y <a href="https://es.wikipedia.org/wiki/Yevgeniy_Landis" class="link  white hover-white-60" target="_blank">Yevgeniy Landis</a> . Lo dieron a conocer en la publicación de un artículo en 1962, «An algorithm for the organization of information» («Un algoritmo para la organización de la información»).

Los árboles AVL están siempre equilibrados de tal modo que para todos los nodos, la altura de la rama izquierda no difiere en más de una unidad de la altura de la rama derecha o viceversa. Gracias a esta forma de equilibrio (o balanceo), la complejidad de una búsqueda en uno de estos árboles se mantiene siempre en orden de complejidad log n. El factor de equilibrio puede ser almacenado directamente en cada nodo o ser computado a partir de las alturas de los subárboles.

### Árbol rojo-negro

Un árbol rojo-negro es un tipo abstracto de datos. Concretamente, es un árbol binario de búsqueda equilibrado, una estructura de datos utilizada en informática y ciencias de la computación. La estructura original fue creada por Rudolf Bayer en 1972, que le dio el nombre de “árboles-B binarios simétricos”, pero tomó su nombre moderno en un trabajo de Leo J. Guibas y Robert Sedgewick realizado en 1978. Es complejo, pero tiene un buen peor caso de tiempo de ejecución para sus operaciones y es eficiente en la práctica. Puede buscar, insertar y borrar en un tiempo O(log n), donde n es el número de elementos del árbol.
